# Estructura del proyecto

Este proyecto esta preparado para una landing profesional de TE VES GUAUU y para crecer hacia un futuro e-commerce.

## `src/assets/`

Recursos visuales estaticos usados por la aplicacion.

- `images/`: fotografias, imagenes promocionales y piezas visuales generales.
- `icons/`: iconos propios del proyecto.
- `logos/`: versiones del logo y recursos de identidad visual.

## `src/components/`

Componentes reutilizables de React, organizados por responsabilidad.

- `layout/`: estructura general de pagina, como contenedores, header o footer.
- `sections/`: bloques principales de una landing, como hero, servicios o contacto.
- `ui/`: piezas pequeñas de interfaz, como botones, inputs o tarjetas.
- `common/`: componentes compartidos que no pertenecen a una seccion especifica.

## `src/data/`

Datos estaticos o configuraciones simples de contenido, como servicios, testimonios, productos o enlaces.

## `src/hooks/`

Hooks personalizados de React para reutilizar logica entre componentes.

## `src/lib/`

Integraciones, clientes o configuraciones de librerias externas cuando el proyecto las necesite.

## `src/pages/`

Vistas o paginas principales de la aplicacion. Puede crecer cuando se agreguen rutas o secciones de e-commerce.

## `src/styles/`

Archivos de estilos globales o utilidades CSS propias del proyecto, separados de la configuracion base de Tailwind.

## `src/types/`

Tipos e interfaces de TypeScript compartidos en la aplicacion.

## `src/utils/`

Funciones auxiliares reutilizables, sin dependencia directa de la interfaz.
